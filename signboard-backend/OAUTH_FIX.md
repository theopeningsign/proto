# 🔧 OAuth 인증 오류 해결 가이드

## ❌ 오류 메시지

```
액세스 차단됨: Signboard Data Uploader은(는) Google 인증 절차를 완료하지 않았습니다
403 오류: access_denied
```

## ✅ 해결 방법

이 오류는 **테스트 사용자로 등록되지 않아서** 발생합니다.

### 해결 단계

1. **Google Cloud Console 접속**
   ```
   https://console.cloud.google.com/
   ```

2. **프로젝트 선택**
   - 상단에서 본인이 만든 프로젝트 선택

3. **OAuth 동의 화면으로 이동**
   - 왼쪽 메뉴: **"API 및 서비스"** → **"OAuth 동의 화면"** 클릭

4. **테스트 사용자 추가**
   - 페이지 하단의 **"테스트 사용자"** 섹션 찾기
   - **"+ 사용자 추가"** 버튼 클릭
   - 본인 이메일 주소 입력: `hyukjune.gp@gmail.com`
   - **"저장"** 클릭

5. **업로드 스크립트 다시 실행**
   ```powershell
   cd D:\cursor\proto\signboard-backend
   
   # 기존 토큰 삭제 (선택사항)
   Remove-Item token.json -ErrorAction SilentlyContinue
   
   # 다시 실행
   python upload_to_drive.py
   ```

## 📸 스크린샷 가이드

### 테스트 사용자 추가 위치

```
OAuth 동의 화면 페이지
  ↓
스크롤 다운
  ↓
"테스트 사용자" 섹션
  ↓
[+ 사용자 추가] 버튼
  ↓
이메일 입력 → 저장
```

## ⚠️ 주의사항

- **테스트 사용자로 등록된 이메일만** 앱에 접근 가능
- 여러 계정을 사용한다면 모든 이메일 추가 필요
- 최대 100명까지 테스트 사용자 추가 가능

## ✅ 확인 방법

테스트 사용자가 제대로 추가되었는지 확인:

1. OAuth 동의 화면 페이지에서
2. "테스트 사용자" 섹션에 본인 이메일이 표시되는지 확인
3. 이메일이 보이면 정상!
